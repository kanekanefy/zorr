# zorr

A self-hosted private deployment of [`trigonal-bacon/gardn`](https://github.com/trigonal-bacon/gardn),
a florr.io-style multiplayer game. Containerized via Docker, built in CI,
deployed via Dokploy.

## What's in here

| Path | 作用 |
|------|------|
| `upstream/gardn/` | gardn pinned as git submodule (see `UPSTREAM_PIN.txt`) |
| `patches/` | three small cosmetic patches applied at build time |
| `Dockerfile` | multi-stage build (`emscripten/emsdk` → `node:20-alpine`) |
| `docker-compose.yml` | local dev shortcut |
| `.github/workflows/` | builds image on push to main, publishes to GHCR |
| `deploy/` | Dokploy deployment script using its tRPC API |
| `cloudflare/` | runbook for putting CF proxy in front (optional) |
| `docs/superpowers/` | design spec + implementation plan |

## Quick start: local dev

```bash
git clone --recurse-submodules https://github.com/kanekanefy/zorr.git
cd zorr
docker compose up --build
# First build is slow (~10-15 min, downloads emscripten/emsdk image).
# Subsequent builds are cached.
# Open http://localhost:9001
```

## Quick start: production deploy

```bash
# 1. Push code to GitHub → GHA auto-builds & pushes to ghcr.io/kanekanefy/zorr:latest
git push origin main

# 2. Deploy to Dokploy (creates project, app, domain, triggers deploy)
export DOKPLOY_URL=https://dok.inglegames.com
export DOKPLOY_API_KEY=<your-key>
./deploy/dokploy-deploy.sh

# Output ends with: "Once deployed, open: https://<auto-generated>.traefik.me"
```

For Cloudflare-proxied custom domain, see [`cloudflare/README.md`](cloudflare/README.md).

## Updating gardn upstream

```bash
cd upstream/gardn
git fetch && git checkout <new-sha>
cd ../..

# Verify patches still apply
cd upstream/gardn
for p in ../../patches/*.patch; do patch -p1 --dry-run < "$p" || echo "BROKEN: $p"; done

# If broken, edit the .patch files (typically just @@ line numbers / context)
# until dry-run passes.

# Update pin
cd upstream/gardn && PIN=$(git rev-parse HEAD) && cd ../..
echo "Based on gardn commit: $PIN" > UPSTREAM_PIN.txt
git add upstream/gardn UPSTREAM_PIN.txt patches/
git commit -m "chore: bump gardn to $PIN"
```

## What's not in here

- Gameplay changes (no new mobs / petals / balance tweaks)
- Native uWebSockets server (the WASM-Node mode is simpler to containerize;
  native is the documented future perf upgrade)
- Persistence / leaderboards beyond what gardn itself provides
